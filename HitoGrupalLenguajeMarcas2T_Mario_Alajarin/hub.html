XML y JSON

Historia, normas de uso y sintaxis y aplicaciones en el ámbito empresarial

XML

Historia

XML es un lenguaje que viene inventado en los años 70 por IBM para ayudar a las empresas a almacenar grandes cantidades de información. El lenguaje de IBM se denominaba GML (General Markup Language). Había demasiada información y por ello crearon el lenguaje GML para clasificar los datos de manera que se pudieran procesar adecuadamente.

Dicho lenguaje gustó mucho a la gente de ISO, los cuales trabajaban normalizando cuantas cosas pudieras imaginar en la época. Por el año 86, normalizaron el lenguaje GML el cual pasó a llamarse SGML debido a que crearon e implementaron un sistema en inglés.

En el año 89, se creó el lenguaje HTML, el cual fue adoptado tanto por la comunidad como por un gran número de empresas que empezaron a utilizar este lenguaje para dar forma a sus páginas web.

Desde el año 96 hasta la actualidad, se creó una herramienta llamada W3C creada para poner orden al lenguaje HTML lo cual falló ya que HTML se descontroló.

Esta misma herramienta W3C empezó y continúa en la evolución del lenguaje XML. Esta herramienta fué creada para solucionar problemas relacionados con el lenguaje HTML. Algunos problemas eran que el contenido se mezclaba con los estilos que se pretendían aplicar, a veces no se permitía compartir información a dispositivos móviles u ordenadores o que la presentación varía dependiendo del visor que se utilice.

Reglas

    • XML es un lenguaje que requiere unas reglas de sintaxis específicas:

  • Todo elemento dentro del XML necesita tener una etiqueta de cierre.
  • Los elementos de un XML son muy sensibles a mayúsculas y minúsculas.
  • Dentro de un XML los elementos deben estar correctamente anidados.
  • Todo XML debe contener un elemento raíz.
  • Los atributos siempre deben ir entre comillas.

Aplicaciones

    Los documentos XML se utilizan en diversas aplicaciones tales como:

  • Intercambio de información: Los documentos XML se utilizan entre las aplicaciones para el intercambio de información. Esto hace que exista una fácil integración de diferentes sistemas.
  • Documentación electrónica: El XML se utiliza para crear documentos electrónicos como facturas, contratos o formularios que tienen una estructura y contenidos definidos.
  • Integración de aplicaciones: XML permite la integración entre aplicaciones y sistemas de diferentes plataformas lo que le da mucha flexibilidad y escalabilidad.
  • Aplicaciones web: XML se utiliza para la creación de páginas o sitios web como el almacenamiento o recuperación de datos.
  • Análisis de los datos: Los documentos XML se utilizan para recopilar, analizar y presentar los datos o la información. Esto permite una mejor toma de decisiones y una mayor eficiencia durante los procesos.

JSON

Historia

JSON (JavaScript Object Notation) es un archivo que ordena los datos de manera legible y organizada. Los datos en el archivo JSON se presentan en archivos de texto que van entre pares de llaves o claves.

Al tener una gran relación con JavaScript, este lenguaje puede ser leído por varios lenguajes de programación lo que hace que debido a la universalidad de JavaScript haya logrado JSON un gran prestigio. Muchos sitios web utilizan JSON para compartir datos en tiempo real por Internet. Por esta razón, los feeds de JSON son más utilizables que los feeds XML.

JSON fué creado por Douglas Crockford quien sigue siendo la figura principal para mantener el sitio web json.org. La primera especificación de JSON surgió de ECMA-404 de 2013 pero el sitio web de JSON fué creado en 2002. Aplicaciones como Yahoo o Google comenzaron a utilizarlo desde 2005 y 2006.

JSON en otras palabras JSON funciona un poco como XML (pero menos desarrollado) y facilita como la información se estructura dentro de la computadora. JSON se utiliza solo para transferir datos por lo cual no se aceptan comentarios. Si el documento está escrito en JavaScript los comentarios serían aceptados. JSON es un lenguaje comprensible tanto para la mente humana como para la máquina.

JavaScript Object Notation son solo textos por lo tanto se pueden almacenar de muchas maneras. Se pueden almacenar estos datos en bases de datos, en archivos de textos separados o en el propio archivo JSON.

Reglas

    JSON se utiliza en varias situaciones como:

  • Solicitudes AJAX, ya que JavaScript es más rápido y fácil de explorar qué XML.
  • Para enviar información de un usuario a una página web y viceversa ya que lo convierte en un lenguaje ideal y universal para el intercambio de datos.
  • Formato de texto para implementar tags de datos estructurados de schema.org, que estructuran las páginas HTML y permiten que los mecanismos de búsqueda entiendan el contenido. Actualmente, Google también recomienda este formato para optimizar estrategias de SEO.

    • Además JSON requiere algunas normas de sintaxis como:

  • Todos los datos a representar deben estar entre llaves { } excepto cuando se refiera a un array que va con los corchetes [ ].
  • En los artículos JSON las comillas simples no están permitidas.
  • La llave JSON tiene que ser simple e ir entre comillas dobles.
  • Los números no deben ir entre comillas dobles porque se transformarian en string.
  • El tipo de dato null no debe ir entre comillas dobles.
  • Los valores booleanos sólo pueden ser verdaderos o falsos.
  • Cada par llave:valor debe terminar con una coma, excepto el último elemento
  • Un solo objeto dentro de un arreglo debe terminar también con una coma.

Aplicaciones

    • A continuación podemos observar algunas aplicaciones de JSON en el ámbito empresarial:

  • Integración de aplicaciones: Un archivo JSON permite intercambiar datos de manera fácil y eficiente. Por ejemplo, una aplicación de gestión de inventarios utiliza JSON para actualizar el registro de ventas e inventario.
  • Servicios web: JSON es un formato que permite la transferencia de datos en sitios web. Por ejemplo, una empresa puede utilizar JSON para acceder a datos de otras aplicaciones o para compartir información con terceros.
  • Almacenamiento de datos: JSON es un formato flexible y fácil de leer. Por ejemplo puedes usar archivos JSON para almacenar datos sobre clientes, ventas u otra información importante.
  • Comunicación de datos en tiempo real: JSON permite el intercambio de información en tiempo real en chats en línea, videojuegos o sistemas de monitoreo.
